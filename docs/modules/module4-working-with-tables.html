6.1 1 - Summarizing a Table with group_by() and summarise()

A constant feature of statistical analysis is the presence of groups or categories that divide the sample into subsets. This process is often referred to as grouping. Variables used to separate observations into groups are called factors in R.

For example, the diamonds dataset introduced in Module 3 contains three factor variables:

  • cut: the quality of the diamond cut, ranging from Fair to Ideal (5 levels)

  • color: the color grade, from D to J (7 levels)

  • clarity: the clarity grade, from I1 to IF (8 levels)

To obtain the list of levels for a factor, use the levels() function.

levels(diamonds$cut)
[1] "Fair"      "Good"      "Very Good" "Premium"   "Ideal"

The group_by() function, combined with summarise(), allows you to perform calculations separately for each subgroup of a dataset.

For example, we can calculate the average diamond price as a function of clarity in two steps:

  1. Declare the grouping factor (here, clarity) in the group_by() function.

  2. Declare the variable and the summary function inside summarise(): here price and mean(price).

# Average diamond price by clarity

diamonds %>% group_by(clarity) %>%
  summarise(Mean = mean(price))

# Mean, median, and standard deviation of price by clarity

diamonds %>% group_by(clarity) %>%

But we can go even further. The mutate() function is sensitive to group_by(), meaning that calculations can be performed either on the entire dataset or separately within each group.

Suppose we want to standardize diamond prices within each cut category. We can proceed as follows:

  1. Calculate the mean and standard deviation for each group and add them to the dataset using mutate().

  2. Compute standardized prices for each diamond.

# Standardizing prices by cut category

diamonds %>% group_by(cut) %>%
  mutate(Mean = mean(price),
         "Standard deviation" = sd(price)) %>%
  ungroup() %>%
  mutate(centered_price = price - Mean,
         standardized_price = centered_price / `Standard deviation`) %>%
  select(cut, price, Mean, `Standard deviation`,
         centered_price, standardized_price) %>%
  arrange(cut)

6.2 2 - Reshaping a Table with Pivot Functions

A standard data file contains one observation per row and one variable per column. This format is not always optimal for analysis. Spreadsheet software often provides a feature called a pivot table, which allows summary tables to be generated. In R, this functionality is provided by two functions:

  • pivot_wider(): creates as many columns as there are levels of a factor. The number of columns increases and the table becomes wider.

  • pivot_longer(): combines multiple columns into two variables, one containing the variable names and the other containing their values. The number of columns decreases while the number of rows increases, making the table longer.

The following example illustrates the difference. Here is a dataset in long format.

The same data can be converted into wide format by pivoting on the Year variable. The single Year column is transformed into multiple columns corresponding to each year, while the values from the Sales variable are used to populate the new table.

Let us create this example in R. To store it in memory for later use, we assign it to a variable called long using the assignment operator <-. The tibble() function creates a data frame in R.

# Create and assign a sample table

long <- tibble(
  Product = c("A", "A", "B", "B"),
  Year = c(2020, 2021, 2020, 2021),
  Sales = c(100, 150, 200, 250)
)

long
# A tibble: 4 × 3
  Product  Year Sales
  <chr>   <dbl> <dbl>
1 A        2020   100
2 A        2021   150
3 B        2020   200
4 B        2021   250

This table is now stored in memory and can be found in the Environment pane of RStudio.

6.2.1 Converting a Long Table into a Wide Table

# Convert from long format to wide format

wide <- long %>%
  pivot_wider(
    names_from = "Year",
    values_from = "Sales"
  )

print(wide)
# A tibble: 2 × 3
  Product `2020` `2021`
  <chr>    <dbl>  <dbl>
1 A          100    150
2 B          200    250

6.2.2 Converting a Wide Table into a Long Table

# Convert from wide format to long format

wide %>% pivot_longer(
  cols = -Product,
  names_to = "Year",
  values_to = "Sales"
)
# A tibble: 4 × 3
  Product Year  Sales
  <chr>   <chr> <dbl>
1 A       2020    100
2 A       2021    150
3 B       2020    200
4 B       2021    250
